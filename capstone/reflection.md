#Reflection - Questions

##What are the three Git states?
The three Git states are as follows;
**Working Directory:** This location stores files which can be still edited.
**Staging Area:**  This location serves as a preparatory area for files that are to be committed to a repository.
**Repository:** This is the permanent location for files that have been staged.

##Why is Git useful?
Git is useful because it helps developers to easily track changes they've made to project files, making it easy for developers to experiment safely. Git also secures a developers work.

##Why do commit messages matter?
Every project needs proper documentation. Commit messages matter in the sense that they give a clear explanation of what changes were made to a file and why. When you commit changes with messages, it is 
easier for an outsider to quickly understand all changes you've made without having to go through your code.
